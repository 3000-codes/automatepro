# 日志窗口系统需求规格说明书

## 一、模块概述

### 1.1 模块背景

日志窗口系统是 AutoMate Pro 自动化工具的核心监控组件，负责实时展示应用运行状态、点击操作记录和系统事件。本模块旨在为用户提供一个功能完善、性能优秀的日志查看和管理工具，满足开发调试和日常使用双重场景的需求。

当前项目已完成 MVP 版本的基础日志功能，包括基本的日志显示和独立窗口支持。本文档将在现有基础上定义标准版日志系统的完整需求，为后续开发提供详细指导。

### 1.2 模块目标

本模块的核心目标是构建一个功能完备的日志窗口系统，具体包括：

**实时性目标**：实现日志消息的实时推送和显示，确保用户能够即时获取最新的运行状态信息。日志从产生到显示的延迟应控制在 100 毫秒以内，保证事件追踪的时效性。

**功能性目标**：提供多级别日志过滤、关键词搜索、日志导出等标准版功能。支持按日志级别、时间和内容进行多维度过滤，使用户能够快速定位感兴趣的信息。

**可用性目标**：提供清晰直观的用户界面，支持日志级别颜色区分、时间戳显示、消息格式化等功能。确保日志窗口在不同使用场景下都具备良好的可读性和操作性。

**持久性目标**：实现可配置的日志持久化存储机制，允许用户控制是否保存日志、保存路径和保存格式。支持手动导出和自动保存两种模式。

## 二、功能需求

### 2.1 日志记录功能

#### 2.1.1 日志级别定义

系统支持四级日志级别，每个级别对应不同的重要程度和显示样式：

**DEBUG（调试级）**：用于开发调试过程中的详细信息，包括变量值、执行路径、函数调用等。此级别日志仅在调试模式下显示，生产环境下默认关闭。DEBUG 级别日志采用灰色文本显示，便于与生产日志区分。

**INFO（信息级）**：用于记录正常的业务流程和操作事件，包括点击开始、点击停止、配置加载、预设切换等。INFO 级别是默认的日志级别，记录应用正常运行时的关键事件。INFO 级别日志采用默认主题色显示。

**WARN（警告级）**：用于记录潜在问题和异常情况，但不影响核心功能运行。例如热键冲突提示、权限不足警告、配置参数异常等。WARN 级别日志采用黄色或橙色文本显示，提示用户注意。

**ERROR（错误级）**：用于记录功能异常和错误事件，包括点击执行失败、配置解析错误、系统调用异常等。ERROR 级别日志采用红色文本显示，并应提供清晰的错误描述和可能的解决建议。

#### 2.1.2 日志内容规范

每条日志记录应包含以下标准字段：

**时间戳**：精确到毫秒级的发生时间，格式为「时:分:秒.毫秒」（如 14:35:22.156）。时间戳应使用本地时区，并在设置中提供 12/24 小时制选项。

**日志级别**：该条日志的级别标识，采用上述四级分类之一。

**消息内容**：日志的正文信息，应简洁清晰地描述事件。消息长度无硬性限制，但建议单条消息不超过 500 字符，超长内容自动截断或折叠。

**来源模块**（可选）：产生日志的模块标识，如 ClickEngine、HotkeyService、PresetManager 等。便于快速定位日志来源。

**附加数据**（可选）：与日志相关的额外信息，如坐标数据、配置参数、错误堆栈等。附加数据默认折叠显示，用户可展开查看详情。

#### 2.1.3 日志事件类型

系统需要记录以下类型的日志事件：

**引擎事件**：点击引擎的启动、停止、暂停、恢复状态变化；点击操作的执行记录，包括坐标、按钮、次数、实际间隔；引擎初始化和配置加载结果。

**配置事件**：当前配置参数的变更记录；预设的保存、加载、删除操作；用户偏好设置的修改。

**系统事件**：窗口状态变化（显示、隐藏、最小化）；全局热键的注册和响应；应用前台焦点变化。

**错误事件**：所有异常和错误的详细记录；系统权限相关的问题提示；第三方服务调用失败信息。

### 2.2 日志显示功能

#### 2.2.1 主显示区域

日志显示区域是窗口的核心内容区，采用以下布局和交互设计：

**列表视图**：日志以垂直列表形式展示，最新日志自动滚动到顶部（可配置）。每条日志占用一行显示不下沉，内容过长时自动换行或显示省略号。提供虚拟滚动支持，确保在大量日志下的流畅滚动性能。

**自动滚动**：默认启用自动滚动功能，新日志产生时自动滚动到底部查看最新内容。当用户手动向上滚动查看历史日志时，自动滚动暂停并显示「已暂停自动滚动」提示。手动返回到底部时恢复自动滚动。

**时间间隔标记**：当日志时间间隔超过指定阈值（如 5 秒）时，显示时间分隔线，标注相隔的时间。便于用户了解日志的时间分布。

#### 2.2.2 样式与格式

**颜色方案**：不同日志级别采用不同颜色区分。DEBUG 级使用灰色（#9E9E9E），INFO 级使用主题色（蓝色或随主题变化），WARN 级使用警告色（#FF9800），ERROR 级使用错误色（#F44336）。颜色应符合 WCAG 对比度要求，确保可读性。

**文本样式**：日志内容采用等宽字体（Monospace）显示，确保对齐整齐。消息文本使用常规字重，级别标签使用中等字重加粗。时间戳使用比消息稍淡的颜色显示。

**图标标识**：每条日志级别可显示对应的图标标识。DEBUG 显示调试图标，INFO 显示信息图标，WARN 显示警告图标，ERROR 显示错误图标。图标放置在日志行左侧。

#### 2.2.3 详细信息展示

**折叠/展开**：包含附加数据的日志行默认折叠附加部分，仅显示主要消息。用户点击日志行或展开按钮可展开查看完整信息。展开状态在会话期间保持。

**右键菜单**：选中日志行后右键点击，显示上下文菜单，提供以下操作：复制该条日志、复制完整信息（含时间戳和级别）、查看详情对话框。

**详情对话框**：双击日志行打开详情对话框，完整展示该条日志的所有字段。详情对话框支持复制完整内容和关闭操作。

### 2.3 日志过滤功能

#### 2.3.1 级别过滤

**多选过滤器**：提供四个复选框或按钮，分别对应 DEBUG、INFO、WARN、ERROR 四个级别。用户可以勾选一个或多个级别，未勾选的级别日志将隐藏。默认勾选 INFO、WARN、ERROR，关闭 DEBUG。

**快捷按钮**：提供「全部显示」「仅错误」「仅警告+错误」等快捷过滤器。点击后自动设置对应的级别复选框状态。

**记住设置**：过滤器状态在会话期间保持，重新打开日志窗口时恢复上次设置。

#### 2.3.2 关键词搜索

**搜索框**：提供文本输入框作为搜索入口，支持实时过滤。输入关键词后，日志列表立即更新为包含该关键词的日志。搜索匹配不区分大小写。

**高亮显示**：搜索关键词在日志内容中高亮显示，使用黄色背景或其他显眼样式。多关键词时同时高亮所有匹配项。

**结果计数**：搜索框旁边显示匹配结果数量，如「找到 23 条记录」。无匹配结果时显示「无匹配」提示。

**清除按钮**：搜索框右侧提供清除按钮，点击后清空搜索内容并恢复显示全部日志。

#### 2.3.3 时间范围过滤

**快速选择**：提供「全部」「最近 5 分钟」「最近 1 小时」「今天」等快速时间范围选项。点击后自动设置时间过滤条件。

**自定义范围**：提供自定义时间范围选择器，用户可以指定开始时间和结束时间。支持日期和时间组合选择。

**清除过滤**：提供「清除时间过滤」按钮，点击后移除时间范围限制。

### 2.4 日志导出功能

#### 2.4.1 导出格式

**纯文本格式（.txt）**：每条日志一行，格式为「时间戳 [级别] 消息内容」。附加数据单独成行，缩进显示。适合查看和打印。

**JSON 格式（.json）**：结构化输出，每条日志作为一个 JSON 对象，包含时间戳、级别、消息、来源、附加数据等字段。适合程序解析和处理。

**CSV 格式（.csv）**：逗号分隔格式，第一行为表头，后续每行一条日志。适合导入 Excel 等表格软件分析。

#### 2.4.2 导出选项

**范围选择**：用户可以选择导出的日志范围。选项包括：仅当前视图（考虑过滤条件）、全部日志、自定义时间范围。

**编码选择**：提供编码格式选择，默认为 UTF-8。兼容旧系统可选 GBK 编码。

**文件名**：导出文件默认命名为「automatepro_日志_日期时间.格式」。用户可以自定义文件名。

#### 2.4.3 导出操作

**文件保存对话框**：点击导出后弹出系统文件保存对话框，用户选择保存位置和文件名。默认保存到用户文档目录或上次使用的目录。

**复制到剪贴板**：提供「复制到剪贴板」选项，将当前视图日志（考虑过滤条件）复制为纯文本格式。复制完成后显示「已复制到剪贴板」提示。

### 2.5 日志管理功能

#### 2.5.1 清空日志

**清空按钮**：提供「清空日志」按钮，点击后清空当前显示的所有日志。清空前弹出确认对话框「确定要清空所有日志吗？此操作不可撤销。」

**自动清空选项**：在设置中提供「启动时自动清空日志」选项。启用后，每次应用启动时自动清空上次运行的日志记录。

#### 2.5.2 日志数量限制

**内存限制**：为防止内存占用过高，设置内存中保留的最大日志数量，默认 10000 条。超过限制时，自动删除最早的日志。

**数量显示**：窗口状态栏显示当前日志数量和限制，如「日志: 3,245 / 10,000」。接近限制时（如 90%）显示警告色提示。

**限制配置**：在设置中提供日志数量限制的配置选项，允许用户调整限制值。最小 1000 条，最大 100000 条。

### 2.6 持久化存储功能

#### 2.6.1 存储配置

**启用开关**：在设置页面提供「保存日志到文件」开关。关闭时日志仅保存在内存中，退出应用后丢失。默认设置为关闭状态。

**保存路径**：用户可以配置日志文件的保存目录。提供「浏览」按钮打开系统文件夹选择对话框。默认保存到用户文档目录下的 AutoMatePro/Logs 文件夹。

**文件命名**：支持配置日志文件的命名规则。选项包括：「按日期命名」（每天一个文件）、「按启动命名」（每次启动一个文件）、「固定名称」。默认「按日期命名」。

#### 2.6.2 自动保存

**保存间隔**：提供自动保存间隔配置，选项包括：「实时保存」（每次日志产生即保存）、「每分钟保存」「每 5 分钟保存」「退出时保存」。默认「每分钟保存」。

**文件轮转**：当选择「按日期命名」时，每天一个日志文件，历史文件保留。可配置保留天数，如「保留 7 天」「保留 30 天」「永久保留」。默认保留 7 天。

#### 2.6.3 手动保存

**保存当前日志**：提供「立即保存」按钮，点击后将当前所有日志保存到指定文件。保存完成后显示「日志已保存到: [路径]」提示。

## 三、非功能需求

### 3.1 性能需求

#### 3.1.1 显示性能

**滚动流畅度**：在 10000 条日志记录下，滚动操作应保持 60fps 的流畅度，不出现卡顿。采用虚拟滚动技术，只渲染可视区域内的日志项。

**渲染延迟**：从日志产生到显示在屏幕上的延迟不超过 100 毫秒。对于高频日志（如 100 CPS 的点击记录），应批量渲染而非逐条渲染，避免阻塞主线程。

**内存占用**：日志窗口的内存占用应与日志数量成正比。10000 条日志的内存占用不超过 50MB。确保长时间运行不会导致内存泄漏。

#### 3.1.2 过滤性能

**搜索响应**：关键词搜索的响应时间应控制在 200 毫秒以内。即使用户快速输入多个字符，也应在完成输入后 200ms 内显示搜索结果。

**切换延迟**：切换日志级别过滤时，视图更新应在 100ms 内完成。避免明显的等待感。

### 3.2 可用性需求

#### 3.2.1 界面布局

**窗口尺寸**：日志窗口默认尺寸为 600x400 像素，最小尺寸为 400x300 像素。支持调整窗口大小，应保存窗口尺寸设置。

**布局分区**：窗口分为三个区域：顶部工具栏（过滤和操作按钮）、中部日志列表（主显示区域）、底部状态栏（日志数量和状态信息）。

**响应式设计**：窗口尺寸变化时，内容区域自适应调整。工具栏在窗口过窄时允许折行显示。

#### 3.2.2 键盘操作

**快捷键支持**：提供以下键盘快捷键：Ctrl+F 聚焦搜索框、Ctrl+L 清空日志、Ctrl+S 导出日志、Escape 清除搜索内容或关闭对话框、上下箭头选择日志行。

**焦点管理**：Tab 键在工具栏各项之间切换焦点，确保仅使用键盘也能完成所有操作。

#### 3.2.3 主题适配

**浅色主题**：在浅色主题下，日志各级别颜色清晰可辨，背景为白色或浅灰色。

**深色主题**：在深色主题下，日志各级别颜色适当调整以确保对比度，背景为深灰色或黑色。

**跟随系统**：支持跟随系统主题变化自动切换，无需重启应用。

### 3.3 国际化需求

#### 3.3.1 界面文本

所有用户界面文本必须支持国际化，包括：工具栏按钮文本、对话框标题和内容、提示消息和状态文本、设置选项标签。

#### 3.3.2 日期时间格式

时间戳格式应适配不同地区习惯：中文环境默认「时:分:秒.毫秒」格式、英文环境可选择「HH:mm:ss.ms」或「hh:mm:ss a」格式。

## 四、技术方案

### 4.1 数据结构设计

#### 4.1.1 日志条目模型

```dart
class LogEntry {
  final DateTime timestamp;    // 时间戳
  final LogLevel level;       // 日志级别
  final String message;       // 主消息
  final String? source;       // 来源模块
  final Map<String, dynamic>? extra;  // 附加数据
}
```

#### 4.1.2 日志级别枚举

```dart
enum LogLevel {
  debug,
  info,
  warn,
  error,
}
```

### 4.2 状态管理设计

#### 4.2.1 日志状态

使用 Riverpod 管理日志状态：

```dart
// 日志列表状态
final logListProvider = StateNotifierProvider<LogListNotifier, List<LogEntry>>;

// 过滤条件状态
final logFilterProvider = StateNotifierProvider<LogFilterNotifier, LogFilter>;

// 日志设置状态
final logSettingsProvider = StateNotifierProvider<LogSettingsNotifier, LogSettings>;
```

#### 4.2.1 日志服务接口

```dart
abstract class LogService {
  void debug(String message, {String? source, Map<String, dynamic>? extra});
  void info(String message, {String? source, Map<String, dynamic>? extra});
  void warn(String message, {String? source, Map<String, dynamic>? extra});
  void error(String message, {String? source, Map<String, dynamic>? extra, Object? error, StackTrace? stackTrace});
}
```

### 4.3 持久化方案

#### 4.3.1 文件存储

使用 path_provider 获取标准目录，使用 dart:io 进行文件操作。日志文件采用追加写入模式，避免频繁 IO 影响性能。

#### 4.3.2 配置存储

使用 Hive 存储日志相关设置，包括：启用持久化开关、保存路径、文件命名规则、自动保存间隔、保留天数、最大日志数量。

## 七、验收标准

### 7.1 功能验收

| 功能 | 验收条件 |
|------|----------|
| 日志显示 | 日志能够实时显示在窗口中，时间戳精确到毫秒 |
| 级别过滤 | 切换级别复选框，日志列表正确更新显示 |
| 关键词搜索 | 输入关键词，日志列表立即过滤并高亮匹配内容 |
| 日志导出 | 导出 TXT/JSON 文件，内容完整可读 |
| 持久化存储 | 启用保存后，日志按配置保存到指定目录 |

### 7.2 性能验收

| 指标 | 验收条件 |
|------|----------|
| 滚动流畅度 | 10000 条日志下滚动保持 60fps |
| 搜索响应 | 搜索响应时间小于 200ms |
| 内存占用 | 10000 条日志内存占用小于 50MB |
| 渲染延迟 | 日志产生到显示延迟小于 100ms |

### 7.3 可用性验收

| 场景 | 验收条件 |
|------|----------|
| 浅色主题 | 日志各级别颜色清晰可辨 |
| 深色主题 | 日志各级别颜色符合对比度要求 |
| 窗口调整 | 改变窗口尺寸，布局自适应 |
| 主题切换 | 切换系统主题，日志窗口跟随变化 |
