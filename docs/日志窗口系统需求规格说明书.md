# 日志窗口系统需求规格说明书

## 一、模块概述

### 1.1 模块背景

日志窗口系统是 AutoMate Pro 自动化工具的核心监控组件，负责实时展示应用运行状态、点击操作记录和系统事件。本模块旨在为用户提供一个功能完善、性能优秀的日志查看和管理工具，满足开发调试和日常使用双重场景的需求。

当前项目已完成日志窗口系统的开发，实现了所有核心功能。本文档记录已完成的需求和实现状态，为后续维护和迭代提供参考。

### 1.2 模块目标

本模块的核心目标是构建一个功能完备的日志窗口系统，具体包括：

**实时性目标**：实现日志消息的实时推送和显示，确保用户能够即时获取最新的运行状态信息。日志从产生到显示的延迟应控制在 100 毫秒以内，保证事件追踪的时效性。

**功能性目标**：提供多级别日志过滤、关键词搜索、日志导出等标准版功能。支持按日志级别、时间和内容进行多维度过滤，使用户能够快速定位感兴趣的信息。

**可用性目标**：提供清晰直观的用户界面，支持日志级别颜色区分、时间戳显示、消息格式化等功能。确保日志窗口在不同使用场景下都具备良好的可读性和操作性。

**持久性目标**：实现可配置的日志持久化存储机制，允许用户控制是否保存日志、保存路径和保存格式。支持手动导出和自动保存两种模式。

## 二、功能需求

### 2.1 日志记录功能 

#### 2.1.1 日志级别定义 

系统支持四级日志级别，每个级别对应不同的重要程度和显示样式：

- **DEBUG（调试级）**：用于开发调试过程中的详细信息。灰色文本显示。
- **INFO（信息级）**：用于记录正常的业务流程和操作事件。默认主题色显示。
- **WARN（警告级）**：用于记录潜在问题和异常情况。黄色/橙色文本显示。
- **ERROR（错误级）**：用于记录功能异常和错误事件。红色文本显示。

#### 2.1.2 日志内容规范 

每条日志记录包含以下标准字段：

- **时间戳**：精确到毫秒级的发生时间，格式为「时:分:秒.毫秒」
- **日志级别**：该条日志的级别标识
- **消息内容**：日志的正文信息
- **来源模块**（可选）：产生日志的模块标识
- **附加数据**（可选）：与日志相关的额外信息

#### 2.1.3 日志事件类型 

系统记录以下类型的日志事件：

- **引擎事件**：点击引擎的启动、停止、状态变化
- **配置事件**：配置参数变更、预设操作
- **系统事件**：窗口状态变化、全局热键响应
- **错误事件**：异常和错误详细记录

### 2.2 日志显示功能 

#### 2.2.1 主显示区域 

- **列表视图**：日志以垂直列表形式展示，最新日志在底部
- **自动滚动**：默认启用自动滚动，手动向上滚动时暂停
- **时间戳显示**：每条日志显示精确到毫秒的时间戳

#### 2.2.2 样式与格式 

- **颜色方案**：不同级别采用不同颜色区分
- **文本样式**：采用等宽字体显示
- **图标标识**：每条日志显示级别对应的图标

#### 2.2.3 详细信息展示 

- **详情对话框**：双击日志行打开详情对话框
- **右键菜单**：提供复制操作
- **折叠/展开**：附加数据可展开查看

### 2.3 日志过滤功能 

#### 2.3.1 级别过滤 

- **多选过滤器**：四个复选框对应四个级别
- **记住设置**：过滤器状态在会话期间保持

#### 2.3.2 关键词搜索 

- **搜索框**：支持实时过滤
- **高亮显示**：搜索关键词高亮显示
- **清除按钮**：一键清空搜索内容

#### 2.3.3 时间范围过滤 

- **快速选择**：「全部」「最近 5 分钟」「最近 1 小时」「今天」

---

### 2.4 日志导出功能 

#### 2.4.1 导出格式     

- **TXT 格式**：每条日志一行，格式为「时间戳 [级别] 消息内容」
- **JSON 格式**：结构化输出，包含所有字段
- **CSV 格式**：逗号分隔格式

#### 2.4.2 导出操作 

- **文件保存**：保存到用户文档目录
- **复制到剪贴板**：一键复制当前日志

---

### 2.5 日志管理功能 

#### 2.5.1 清空日志 

- **清空按钮**：提供清空功能，带确认对话框

#### 2.5.2 日志数量限制 

- **内存限制**：默认 10000 条
- **数量显示**：状态栏显示日志数量

### 2.6 窗口管理功能 

#### 2.6.1 独立窗口 

- **游离窗口**：日志窗口独立于主窗口运行
- **单例模式**：只允许打开一个日志窗口
- **拖拽移动**：支持拖拽移动窗口位置
- **最小化/最大化**：提供最小化和最大化按钮
- **大小调整**：可手动调整窗口大小

#### 2.6.2 窗口控制 

- **默认开启**：可在设置中配置启动时默认打开
- **透明度控制**：支持 30%-100% 透明度调节
- **设置同步**：设置项在设置页面和日志窗口均可调整

#### 2.6.3 生命周期管理 

- **主窗口关闭联动**：主窗口关闭时自动关闭日志窗口
- **状态同步**：窗口打开/关闭状态与设置保持同步

### 2.7 持久化存储功能 

#### 2.7.1 存储配置 

使用 Hive 存储以下设置：

- 窗口默认开启状态
- 窗口透明度
- 窗口位置和大小
- 最大日志数量
- 日志自动保存间隔

#### 2.7.2 设置持久化 

- **启动时恢复**：应用启动时恢复上次窗口设置
- **实时保存**：设置变更自动保存

### 4.5 持久化设计

#### 4.5.1 存储结构

使用 Hive 进行本地持久化存储，主要存储以下数据：

| 存储键 | 数据类型 | 说明 |
|--------|----------|------|
| `log_window_enabled` | bool | 日志窗口是否默认开启 |
| `log_window_opacity` | double | 窗口透明度 (0.3-1.0) |
| `log_window_x` | double | 窗口X坐标 |
| `log_window_y` | double | 窗口Y坐标 |
| `log_window_width` | double | 窗口宽度 |
| `log_window_height` | double | 窗口高度 |
| `log_max_count` | int | 最大日志数量 |
| `log_auto_save_interval` | int | 自动保存间隔(秒) |
| `log_level_debug` | bool | DEBUG级别是否启用 |
| `log_level_info` | bool | INFO级别是否启用 |
| `log_level_warn` | bool | WARN级别是否启用 |
| `log_level_error` | bool | ERROR级别是否启用 |

#### 4.5.2 数据一致性策略

- **启动恢复**：应用启动时从 Hive 读取设置，恢复窗口状态
- **实时保存**：设置变更后立即写入 Hive，确保不丢失
- **默认值兜底**：首次运行时使用默认值初始化存储

### 4.6 接口设计

#### 4.6.1 日志管理器接口

```dart
abstract class LogManager {
  /// 添加日志条目
  void addLog(LogEntry entry);

  void clearLogs();
```

### 4.8 模块初始化顺序

系统启动时按以下顺序初始化各模块：

```
1. SettingsStorage (数据层)
   └─> 加载持久化设置

2. MessageBus (数据层)
   └─> 初始化事件总线

3. LogCache (数据层)
   └─> 初始化日志缓存

4. SettingsManager (业务层)
   └─> 从 SettingsStorage 读取配置
   └─> 应用默认配置

5. ThemeService (业务层)
   └─> 初始化主题服务
   └─> 应用保存的主题

6. LogManager (业务层)
   └─> 初始化日志管理器
   └─> 订阅 MessageBus 事件

7. FilterService (业务层)
   └─> 初始化过滤服务

8. ExportService (业务层)
   └─> 初始化导出服务

9. WindowController (业务层)
   └─> 初始化窗口控制器
   └─> 根据设置决定是否打开窗口

10. LogWindow (表现层)
    └─> 构建用户界面
    └─> 订阅 LogManager 状态
```
