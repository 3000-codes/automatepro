# 跨平台自动点击器开发计划

## 一、项目基本信息

| 项目名称 | 跨平台自动点击器 |
|----------|------------------|
| 项目类型 | 桌面应用程序 |
| 技术框架 | Flutter Desktop |
| 目标平台 | Windows、macOS、Linux |
| 开发周期 | 18周 |

## 二、技术选型

### 2.1 核心技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 开发框架 | Flutter 3.x | 跨平台桌面应用框架 |
| 开发语言 | Dart 3.x | 空安全、类型推断 |
| 状态管理 | Riverpod | 声明式依赖注入 |
| 本地存储 | Hive | 轻量级NoSQL数据库 |
| 键鼠模拟 | bixat_key_mouse | Rust编写，高性能 |
| 图像识别 | OpenCV via FFI | 模板匹配、颜色检测 |

### 2.2 架构模式

采用**分层架构**，自上而下分为：

```
┌─────────────────────────────────────┐
│         表现层 (Presentation)       │  Flutter Widget, BLoC
├─────────────────────────────────────┤
│       业务逻辑层 (Business Logic)    │  点击引擎、脚本引擎、触发器
├─────────────────────────────────────┤
│          领域层 (Domain)             │  领域模型、数据结构
├─────────────────────────────────────┤
│      基础设施层 (Infrastructure)     │  键鼠模拟、系统权限、存储
└─────────────────────────────────────┘
```

## 三、开发阶段规划

### 3.1 第一阶段：原型开发（4周）

**里程碑**：M1 - MVP发布  
**交付内容**：具备基础点击功能的可运行产品

| 周次 | 任务内容 | 交付物 |
|------|----------|--------|
| 第1周 | 项目初始化与环境搭建 | Flutter项目框架、依赖配置、基础导航结构 |
| 第2周 | 基础点击功能开发 | 点击引擎核心、坐标拾取、热键控制 |
| 第3周 | UI完善与配置持久化 | 主界面优化、配置保存、预设管理、**透明效果、运行日志** |
| 第4周 | 测试与优化 | 功能测试、Bug修复、用户文档 |

**功能范围**：

- 单次点击 / 连续点击模式
- 0.1-100 CPS可配置频率
- 固定间隔 / 随机延迟
- 全局热键控制（F9启动、F10停止）
- 点击配置预设保存与加载
- **窗口透明效果（0-100%可调）**
- **运行日志实时显示**
- **窗口置顶功能**

### 3.2 第二阶段：脚本功能（6周）

**里程碑**：M2 - 脚本版  
**交付内容**：支持完整脚本编程的产品

| 周次 | 任务内容 | 交付物 |
|------|----------|--------|
| 第5-6周 | 脚本引擎开发 | 词法分析、语法解析、解释器、运行时 |
| 第7周 | 可视化编辑器 | 拖拽式脚本构建、双向转换、模板库 |
| 第8周 | 高级脚本特性 | 变量、函数、条件判断、循环控制 |
| 第9周 | 脚本管理功能 | 文件CRUD、导入导出、版本管理 |
| 第10周 | 测试与文档 | 全面测试、性能优化、脚本文档 |

**功能范围**：

- 类JavaScript脚本语法
- 可视化/代码双模式编辑
- 变量、函数、数组
- 条件判断、循环结构
- 内置指令集（鼠标、键盘、控制、变量）
- 脚本模板库

### 3.3 第三阶段：智能触发（5周）

**里程碑**：M3 - 智能版  
**交付内容**：具备智能触发功能的产品

| 周次 | 任务内容 | 交付物 |
|------|----------|--------|
| 第11周 | 图像识别基础 | OpenCV集成、屏幕截图、模板匹配 |
| 第12周 | 触发器框架 | 触发器管理器、事件机制、条件组合 |
| 第13周 | 颜色识别 | 颜色拾取、颜色检测、变化检测 |

**功能范围**：

- 模板匹配图像识别
- 颜色拾取与区域检测

## 六、关键技术节点

### 6.1 跨平台坐标系统统一

**难点**：不同OS坐标系统差异、DPI适配、多显示器支持

**方案**：

- 内部标准化坐标（屏幕左上角为原点）
- DPI缩放处理
- 多显示器配置支持

### 6.2 权限获取

| 平台 | 权限要求 | 获取方式 |
|------|----------|----------|
| Windows | UIAccess | 提权或系统设置 |
| macOS | Accessibility | 系统偏好设置引导 |
| Linux | X11/Wayland API | 发行版配置 |

### 6.3 游戏兼容性

**挑战**：DirectInput、全屏模式、作弊检测

**方案**：

- 多层输入模拟方案（标准消息→硬件模拟）
- 行为随机化（随机延迟、位置偏移）
- 持续更新兼容主流游戏
